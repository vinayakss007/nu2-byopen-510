# NuCRM: Detailed "No-Code" Modules Implementation Plan

This plan outlines the addition of four high-impact "No-Code" features, integrated into the NuCRM "Modules" system for granular tenant-level activation and Super Admin control.

---

## Pillar 1: Module - Dynamic Calculated Fields (Formula Engine)
*   **Goal:** Transform NuCRM from a static database into a dynamic application by allowing users to define fields that calculate values automatically.
*   **Examples:** 
    *   `Deal Profit` = `{{value}} - {{cost}}`
    *   `Full Name` = `{{first_name}} + " " + {{last_name}}`
    *   `Days to Close` = `{{close_date}} - {{created_at}}`
*   **Technical Implementation:**
    *   **Schema:** Extend `public.custom_field_defs` with `formula` (TEXT) and `is_calculated` (BOOLEAN).
    *   **Engine:** Implement `lib/formula/engine.ts` using a safe expression parser (like `expr-eval`) to evaluate formulas using entity metadata.
    *   **Automation:** Use a database trigger or middleware in `lib/db/client.ts` to re-calculate and update the `metadata` JSONB column whenever dependent fields are modified.

## Pillar 2: Module - Visual Form Builder (Lead Capture)
*   **Goal:** Provide a drag-and-drop interface for non-technical users to build and embed lead generation forms.
*   **Features:**
    *   **Visual Builder:** A new UI at `/tenant/forms/builder` using `dnd-kit` for reordering fields.
    *   **Field Mapping:** Map form inputs directly to CRM columns (e.g., "Phone" -> `contacts.phone`).
    *   **Embed System:** Generate a `<script>` or `<iframe>` snippet for easy integration into external websites.
    *   **Automation:** Submissions automatically create contacts, trigger "New Lead" notifications, and fire "Form Submitted" workflows.

## Pillar 3: Module - External Webhook Automations (The "Zapier" Bridge)
*   **Goal:** Connect NuCRM to the broader SaaS ecosystem (Zapier, Make, Slack, etc.) without writing code.
*   **Implementation:**
    *   **New Action Type:** Add `send_webhook` to the `lib/automation/engine.ts` action executor.
    *   **Workflow Integration:** Add a "Webhook" node to the `workflow-builder.tsx` canvas.
    *   **Reliability:** Utilize the existing `webhook_deliveries` table and retry cron job to ensure 99.9% delivery success even during external downtime.

## Pillar 4: Module - Industry Templates (Niche Setup Modules)
*   **Goal:** Eradicate "Blank Screen Syndrome" by providing pre-configured CRM setups for specific industries.
*   **Niches:**
    *   **Real Estate Module:** Pre-sets "Property" fields, "Buyer/Seller" tags, and "Escrow" pipelines.
    *   **SaaS Module:** Pre-sets "Subscription" tracking, "Churn Risk" logic, and MRR dashboards.
    *   **Consulting Module:** Pre-sets "Project" tasks, "Billable Rate" fields, and "Proposal" stages.
*   **Mechanism:** A "Setup Wizard" that executes batch `INSERT` operations into `custom_field_defs`, `pipelines`, and `automations` based on the selected industry module.

---

## Super Admin Control & Monetization
*   **Granular Gating:** Super Admins can enable/disable these modules per tenant via the `tenant_modules` table.
*   **Usage Metering:** Track "Form Submissions," "Webhook Calls," and "Calculated Field Runs" via `usage_snapshots`.
*   **Tiered Pricing:** Limit these features by Plan (e.g., "Free: 1 Form, Pro: Unlimited Forms & Webhooks").

---

## Implementation Roadmap
1.  **Phase 1:** Super Admin Limit Enforcement & Formula Engine Core.
2.  **Phase 2:** Visual Form Builder UI & Embed Script.
3.  **Phase 3:** Webhook Action & Workflow Builder Update.
4.  **Phase 4:** Industry Template Library & One-Click Setup Wizard.
